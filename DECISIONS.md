# DECISIONS.md

## Why I Chose This Stack

I chose Node.js with Express.js for the backend because it is lightweight, easy to set up, and works well with JavaScript across the entire application. Using the same language for both frontend and backend made development simpler and easier to understand.

I chose MongoDB Atlas because it provides persistent cloud-based storage and integrates well with Node.js through Mongoose. It also allows flexible document-based data modeling, which suited the event and registration data used in this project.

For the frontend, I used plain HTML, CSS, and JavaScript. The assignment specifically allowed plain JavaScript, and using it helped me focus on understanding the core concepts of API communication, authentication, and DOM manipulation without introducing additional framework complexity.

## One Decision I Made That Wasn't Specified

I chose to display event capacity using both a percentage value and a color-coded progress bar. While the assignment required color coding based on capacity usage, showing the percentage alongside the visual indicator makes it easier for administrators to quickly understand how full an event is.

## One Thing I Would Improve With More Time

If given more time, I would add stronger backend validation and additional event management features. For example, I would validate event dates and capacities more thoroughly, allow administrators to edit or delete events, and improve the overall responsiveness of the user interface for mobile devices.
